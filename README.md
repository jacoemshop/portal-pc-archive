# portal-pc-archive

本项目用于归档与发布多个独立的 HTML 页面。这些页面内容彼此独立，不针对任何特定的域名或网站，主要作为静态资源的存储与访问入口。

## 项目简介

`portal-pc-archive` 是一个 HTML 页面归档仓库，旨在提供一个集中存放与展示多个独立页面的空间。每个页面均采用纯前端技术构建，不依赖后端服务，方便直接通过浏览器访问。

仓库中的页面内容可能涵盖信息展示、工具、文档等不同类型，但彼此之间无强关联，仅通过本仓库进行统一管理与发布。

## 目录说明

```
portal-pc-archive/
├── index.html          # 入口页面，列出所有归档页面链接
├── pages/              # 存放各个独立 HTML 页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── README.md           # 本文件
└── LICENSE             # 许可证文件
```

- `index.html`：仓库的主入口页面，通常包含所有归档页面的目录或导航。
- `pages/`：存放各个独立的 HTML 页面文件，每个文件对应一个单独的内容单元。
- `assets/`：存放所有页面可能共享的静态资源，如样式表、脚本文件、图片等。

## 页面归档说明

- 每个独立页面均以 `.html` 文件形式存放在 `pages/` 目录下，文件命名遵循 `page-xxx.html` 的格式，便于识别与引用。
- 页面内容均为静态 HTML，不包含动态数据请求或后端交互逻辑。
- 页面之间互不依赖，可独立访问与展示。
- 页面可能引用 `assets/` 目录下的公共资源，以保证样式与功能的一致性。

## 维护说明

- 新增页面：将新的 HTML 文件放入 `pages/` 目录，并更新 `index.html` 中的链接列表。
- 修改页面：直接编辑对应文件，注意保持文件结构和资源引用的正确性。
- 删除页面：移除 `pages/` 中的对应文件，并同步更新 `index.html` 中的链接列表。
- 公共资源更新：修改 `assets/` 下的文件时，需检查所有引用该资源的页面是否兼容。
